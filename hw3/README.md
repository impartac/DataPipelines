# Тест производительности форматов данных

## Описание задания

Сравнение производительности различных форматов данных:
- **Parquet** - колоночный формат с компрессией
- **Avro** - строчный бинарный формат
- **CSV** - текстовый табличный формат
- **JSON** - текстовый иерархический формат

## Датасет

**Сущность:** Заказы интернет-магазина (E-commerce Orders)

Реалистичная структура данных включает:
- **Информация о заказе:** ID, дата, статус
- **Информация о клиенте:** ID, имя, email
- **Информация о продукте:** ID, название, категория
- **Финансовые данные:** цена, количество, скидки, налоги
- **Доставка:** адрес, даты доставки
- **Обратная связь:** рейтинг, отзывы

**Размер датасета:** ~10 GB

## Установка зависимостей

## Запуск тестирования

```bash
python format_performance_test.py
```

### Параметры конфигурации

В файле `format_performance_test.py` можно настроить:

```python
TARGET_SIZE_GB = 10  # Целевой размер датасета в GB
NUM_RECORDS = ...    # Количество записей
```

Для быстрого тестирования можно уменьшить количество записей:

```python
NUM_RECORDS = 1_000_000  # 1 миллион записей
```

## Что измеряется

### 1. Объём хранения (MB)
- Размер файла на диске для каждого формата

### 2. Время записи (секунды)
- Время записи всего датасета в файл

### 3. Время чтения (секунды)
- **Full Scan:** Чтение всего датасета
- **Filtered Read:** Выборка данных по условию
- **Aggregation:** Групповая агрегация (сумма продаж по категориям)

## Результаты

После выполнения тестов результаты сохраняются в:
- `test_data/performance_results.csv` - таблица с результатами

Результаты включают:
- Время записи для каждого формата
- Время чтения (полного и частичного)
- Время агрегации
- Размер хранения
- Рейтинг по каждой метрике

## Примечания

- **Parquet** обычно показывает лучшую производительность для аналитических запросов благодаря колоночному хранению и компрессии
- **Avro** эффективен для потоковой обработки и схем данных
- **CSV** прост в использовании, но занимает много места и медленно читается
- **JSON** гибкий, но неэффективен по размеру и скорости

## Функция генерации датасета

Класс `EcommerceDataGenerator` автоматически создает реалистичные данные:

```python
generator = EcommerceDataGenerator(seed=42)
df = generator.generate_batch(num_records=10_000_000)
```

Особенности:
- Детерминированная генерация (seed для воспроизводимости)
- Реалистичные имена, даты, адреса
- Различные категории продуктов
- Nullable поля (опциональные данные)
- Связанные данные (цена × количество = итог)
