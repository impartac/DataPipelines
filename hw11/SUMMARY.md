# 📊 Проект: Универсальные WorkflowTemplate для Argo Workflows

## 🎯 Цель проекта

Разработка универсальных и переиспользуемых WorkflowTemplate для Argo Workflows, которые решают частые задачи оркестрации данных в data pipeline'ах.

---

## 📦 Что создано

### ✅ 5 Универсальных WorkflowTemplate

1. **Data Validation Template** - Валидация данных
   - Проверка схемы, NULL значений, дубликатов
   - Гибкие правила валидации через JSON
   - Детальный отчет о результатах

2. **ETL Processing Template** - Extract-Transform-Load обработка
   - 4 типа трансформаций: filter, aggregate, join, enrich
   - Конвертация форматов (CSV, JSON, Parquet, Avro)
   - Чанковая обработка больших данных

3. **Notification Template** - Мультиканальные уведомления
   - Поддержка Slack, Email, Teams, Webhook
   - Умное форматирование с эмодзи
   - Включение метаданных workflow

4. **Conditional Execution Template** - Условная логика
   - 4 типа условий: threshold, status, regex, range
   - 8 операторов сравнения
   - Гибкое принятие решений

5. **Batch File Processor Template** - Пакетная обработка файлов
   - Параллельная обработка (настраиваемое количество воркеров)
   - 5 операций: analyze, copy, move, compress, transform
   - Fail-fast режим и обработка ошибок

### ✅ Основной Workflow

**Data Processing Pipeline** (`main-workflow.yaml`)
- Объединяет все 5 шаблонов в осмысленный пайплайн
- Полная обработка от валидации до финального уведомления
- Условное выполнение шагов на основе результатов
- Обработка ошибок и retry политики

### ✅ Документация

- **README.md** (2000+ строк) - Полная документация всех шаблонов
- **DEPLOYMENT.md** (800+ строк) - Детальное руководство по развертыванию
- **ARCHITECTURE.md** (600+ строк) - Архитектурные диаграммы

## 🏗️ Архитектура решения

### Принципы дизайна

1. **Универсальность** - Каждый шаблон решает одну задачу, но гибко параметризован
2. **Переиспользуемость** - Шаблоны можно комбинировать в любых workflow
3. **Модульность** - Независимые шаблоны, не зависящие друг от друга
4. **Расширяемость** - Легко добавлять новые типы операций и условий
5. **Production-ready** - Обработка ошибок, retry, мониторинг

### Ключевые возможности

✅ **Гибкая параметризация**
- Все критичные значения вынесены в параметры
- Значения по умолчанию для быстрого старта
- JSON конфигурация для сложных настроек

✅ **Богатые outputs**
- Структурированные выходные параметры
- Артефакты для передачи данных
- Метрики для мониторинга

✅ **Обработка ошибок**
- Fail-fast и continue-on-error режимы
- Retry политики с backoff
- onExit handlers для cleanup

✅ **Условная логика**
- Динамическое принятие решений
- When conditions для управления потоком
- Поддержка сложных условий

## 📈 Примеры использования

### Use Case 1: Daily Data Ingestion Pipeline

```
1. Validate incoming data (Data Validation Template)
2. Check if valid (Conditional Template)
3. Transform and load (ETL Template)
4. Archive original files (Batch Processor Template)
5. Send report (Notification Template)
```

### Use Case 2: Quality Assurance Workflow

```
1. Run data quality checks (Data Validation Template)
2. Calculate quality score (ETL Template)
3. Check if score meets threshold (Conditional Template)
4. Send alert if below threshold (Notification Template)
```

### Use Case 3: Batch File Processing

```
1. Scan directory for new files (Batch Processor Template)
2. Process each file (ETL Template)
3. Validate results (Data Validation Template)
4. Notify about completion (Notification Template)
```

## 📞 Использование

### Минимальная установка (5 минут)

```bash
# 1. Установить Argo Workflows
kubectl create namespace argo
kubectl apply -n argo -f https://github.com/argoproj/argo-workflows/releases/download/v3.5.0/install.yaml

# 2. Применить шаблоны
kubectl apply -f workflow-templates.yaml -n argo

# 3. Запустить пример
kubectl create -f main-workflow.yaml -n argo
```
