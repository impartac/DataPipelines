## 🔧 Основные YAML файлы

### 1. workflow-templates.yaml
**Размер:** ~650 строк  
**Назначение:** Основной файл с 5 универсальными WorkflowTemplate

**Содержание:**
```yaml
✅ 5 WorkflowTemplate:
   1. data-validation-template      (~130 строк)
      - Валидация данных
      - Проверка схемы, NULL, дубликатов
      
   2. etl-processing-template       (~150 строк)
      - ETL обработка
      - 4 типа трансформаций
      
   3. notification-template         (~120 строк)
      - Отправка уведомлений
      - Slack, Email, Teams, Webhook
      
   4. conditional-execution-template (~130 строк)
      - Условное выполнение
      - 4 типа условий, 8 операторов
      
   5. batch-file-processor-template (~120 строк)
      - Пакетная обработка файлов
      - 5 операций, параллельная обработка
```

**Как использовать:**
```bash
kubectl apply -f workflow-templates.yaml -n argo
```

### 2. main-workflow.yaml

**Назначение:** Полный data processing pipeline

**Содержание:**
```yaml
✅ Основной Workflow:
   - Использует все 5 шаблонов
   - 8 шагов обработки данных
   - Условное выполнение

   - Retry политики
   - TTL стратегии
   
Шаги:
   1. Pipeline Start Notification
   2. Data Validation
   3. Check Validation Status
   4. ETL Processing (conditional)
   5. Quality Check
   6. Batch File Processing (conditional)
   7. Success Notification
   8. Failure Notification
```

**Как использовать:**
```bash

# или

```
